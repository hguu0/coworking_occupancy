# Техническое задание: Coworking Occupancy Predictor

## 1. Название и концепция
**Название:** Coworking Occupancy Predictor
**Концепция:** Сервис для управления бронированием мест в коворкинге и прогнозирования загруженности помещений. Позволяет пользователям видеть доступные места и бронировать их, а администраторам — анализировать статистику и прогнозировать спрос.

## 2. Целевая аудитория и роли
*   **Гость:** Просмотр списка коворкинг-зон и их текущей доступности (без возможности бронирования).
*   **Участник (Member):** Авторизованный пользователь. Может бронировать места, просматривать историю своих бронирований.
*   **Администратор:** Управление комнатами, просмотр аналитики и прогнозов загруженности.

## 3. Схема данных (Сущности)

### Модель 1: Space (Коворкинг-зона)
Описание помещения или зоны (например, "Open Space", "Meeting Room A").
*   `name` (CharField): Название зоны.
*   `capacity` (IntegerField): Вместимость (количество мест).
*   `description` (TextField): Описание.
*   `price_per_hour` (DecimalField): Цена за час.

### Модель 2: Booking (Бронирование)
Запись о бронировании места пользователем.
*   `user` (ForeignKey -> User): Пользователь, создавший бронь.
*   `space` (ForeignKey -> Space): Забронированная зона.
*   `start_time` (DateTimeField): Начало бронирования.
*   `end_time` (DateTimeField): Конец бронирования.
*   `status` (CharField): Статус (confirmed, cancelled, completed).

### Модель 3: OccupancyLog (Лог загруженности)
Исторические данные для аналитики. Заполняется автоматически или администратором для обучения модели прогнозирования.
*   `space` (ForeignKey -> Space): Ссылка на зону.
*   `timestamp` (DateTimeField): Время фиксации.
*   `occupied_count` (IntegerField): Количество занятых мест.
*   `temperature` (FloatField): Температура воздуха (для корреляции с погодой).
*   `is_holiday` (BooleanField): Является ли день выходным/праздничным.

## 4. Основные сценарии пользователя (User Stories)
1.  **Просмотр доступности:** Пользователь заходит на главную страницу и видит список зон с индикатором текущей загруженности (рассчитывается на основе активных бронирований).
2.  **Бронирование места:** Авторизованный пользователь выбирает зону, дату и время, система проверяет свободные места и создает бронирование.
3.  **Аналитика загруженности:** Администратор просматривает график средней загруженности по дням недели (построенный с использованием Pandas/Matplotlib) для оптимизации цен.

## 5. Технический стек и интеграции
*   **Backend:** Python 3, Django 4.2.
*   **Database:** SQLite (для разработки).
*   **Frontend:** Tailwind CSS (стилизация интерфейса).
*   **Data Analysis:** Pandas (агрегация данных бронирвоаний), Matplotlib (визуализация графиков загрузки).
*   **External Integration:** Mock-up или реальное API погоды (OpenWeatherMap) для обогащения данных `OccupancyLog`.
